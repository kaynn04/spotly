# Implementation Plan: Delete Item

**Branch**: `006-delete-item` | **Date**: 2026-05-06 | **Spec**: [spec.md](./spec.md)

## Summary

Add permanent delete functionality for items in Synop. Users can delete items from a space with a confirmation dialog to prevent accidents. The delete button appears inline next to the Move button for UI consistency. Deletion is permanent (no undo) and the item is immediately removed from the database and UI.

**Performance Goals**: Immediate deletion with sub-200ms latency  
**Constraints**: No cloud sync, no soft delete, permanent deletion only  
**Scale/Scope**: Single-user local-first app, minimal item counts per space (~10-100 items per space)

## Phase 1: Design & Contracts

### 1.1 Data Model

**Item Entity** (existing - no schema changes needed):
```typescript
interface Item {
  id: string;           // UUID
  name: string;         // Item name
  spaceId: string;      // Foreign key to spaces table
  createdAt: string;    // ISO timestamp
}
```

**Database** (existing `items` table - no migration needed):
```sql
CREATE TABLE items (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  space_id TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  FOREIGN KEY (space_id) REFERENCES spaces(id) ON DELETE CASCADE
);
```

**Delete Operation**:
- **Input**: `itemId: string`
- **Output**: Void (no return value)
- **SQL**: `DELETE FROM items WHERE id = ?`
- **Error**: ServiceError with code='DB_ERROR' if deletion fails

### 1.2 Service Layer Contract

**Method to Add**: `ItemService.deleteItem(itemId: string)`

**Signature**:
```typescript
async deleteItem(itemId: string): Promise<void>
```

**Behavior**:
- Accepts item ID
- Calls ItemRepository.deleteItem()
- Throws ServiceError if operation fails
- Returns void on success

**Error Handling**:
```typescript
catch (error) {
  if (error instanceof ServiceError) throw error;
  throw new ServiceError('DB_ERROR', 'Failed to delete item');
}
```

### 1.3 Repository Layer Contract

**Method to Add**: `ItemRepository.deleteItem(itemId: string)`

**Signature**:
```typescript
async deleteItem(itemId: string): Promise<void>
```

**Behavior**:
- Accepts item ID
- Executes parameterized DELETE query
- Throws ServiceError on database error
- Returns void on success

**SQL Pattern**:
```typescript
const statement = await this.db.prepareAsync('DELETE FROM items WHERE id = ?');
await statement.executeAsync(itemId);
await statement.finalizeAsync();
```

### 1.4 UI Integration Points

**Component**: `SpaceDetailScreen` (app/app/space/[id].tsx)

**Changes**:
- Add delete button inline with Move button (item row: [Name] [Move] [Delete])
- Button styling: Red/destructive appearance (e.g., #ff3333)
- Click handler: Show confirmation alert
- Confirmation dialog options: "Delete" (destructive), "Cancel"
- On "Delete": Call ItemService.deleteItem(itemId)
- On success: Refresh item list via loadItems()
- On error: Show error alert

**Error UI**:
```typescript
Alert.alert('Error', 'Failed to delete item. Please try again.');
```

**Success Behavior**:
- Item immediately removed from list
- UI reflects new item count
- No navigation changes

### 1.5 Quickstart

**How to Implement Delete Item**

1. **Add ItemRepository.deleteItem()**
   - Location: `app/src/repositories/ItemRepository.ts`
   - Method: Parameterized DELETE query
   - Error handling: Convert DB errors to ServiceError

2. **Add ItemService.deleteItem()**
   - Location: `app/src/services/ItemService.ts`
   - Validation: None (just pass through to repository)
   - Error handling: Re-throw ServiceError

3. **Add Delete Button to UI**
   - Location: `app/app/space/[id].tsx`
   - Update itemRow FlatList renderItem
   - Add Pressable delete button
   - Add confirmation alert handler
   - Refresh list after deletion

4. **Add Styles**
