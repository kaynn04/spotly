# Implementation Plan: Voice Input

**Branch**: `032-voice-input` | **Date**: May 11, 2026 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/specs/011-voice-input/spec.md`

## Summary

Voice Input adds a floating action button (FAB) to the Synop dashboard. Tapping it activates `expo-speech-recognition` (device-native, no API cost). The user speaks "Add [item] to [space] [container]". A `VoiceParserService` (regex keyword extraction) and `VoiceMatcherService` (fuzzysort fuzzy matching) resolve the spoken words against existing inventory. A confirmation card shows the parsed result before anything is saved. On confirm, `ItemService.createItem()` is called — the existing service, unchanged. No new DB tables. No internet required. Requires an EAS Dev Build (already configured).

**Scale/Scope**: Single user; 5–50 inventory spaces, 5–20 containers per space

## Key Implementation Details

### Permission Flow
- `expo-speech-recognition` handles mic permission request automatically on first use
- If denied: FAB shows disabled state with `opacity: 0.4`; onPress shows an Alert explaining why permission is needed

### Listening State Machine
```
idle → [tap FAB] → listening → [result event] → processing → confirming
                             → [10s timeout]  → error ← [error event]
confirming → [tap Add]    → success → modal closes (loadData() called)
           → [tap Cancel] → modal closes (no data created)
           → [tap Retry]  → listening
error      → [tap Retry]  → listening
           → [tap Cancel] → modal closes
```

### Parser: Anchor Words
- Action verbs: `add | put | place | store | save`
- Prepositions: `to | in | into | inside | at`
- Pattern: `[verb] [item] [preposition] [space] [container?]`

### Matcher: Resolution Priority
1. Exact match (case-insensitive)
2. fuzzysort fuzzy match (top 3 candidates)
3. No match → show picker

### Confirmation Card: Unresolved Fields
- `status: 'exact'` → show name in green (resolved)
- `status: 'fuzzy'` → show suggestion with "Did you mean X?" tap to confirm
- `status: 'none'` → show picker button to manually select
- `'absent'` container → show "Select container" if space has containers, hide if not
- "Add" button disabled until both space and (if applicable) container are resolved

### ItemService Call (existing, no changes)
```typescript
// Item in container:
await ItemService.createItemInContainer(spaceId, containerId, { name: itemName, quantity: 1, description: '' });

// Item in space root:
await ItemService.createItem(spaceId, { name: itemName, quantity: 1, description: '' });
```
