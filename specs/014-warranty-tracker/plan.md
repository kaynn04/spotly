# Implementation Plan: Warranty Tracker

**Branch**: `037-other-tools` | **Date**: 2026-05-13 | **Spec**: [specs/014-warranty-tracker/spec.md](./spec.md)
**Input**: Feature specification from `/specs/014-warranty-tracker/spec.md`

## Summary

Allow users to attach a warranty expiry date to any inventory item. The app schedules two local push notifications (30 days before and on the expiry day) and provides a Warranty Tracker dashboard in the Tools tab grouped by Expiring Soon / Active / Expired. Storage is two new columns on the existing `items` table (Migration 012). Notification scheduling reuses `ReminderService` via a new thin `WarrantyReminderService` wrapper. No new tables, no external APIs.

## Implementation Decisions

All decisions are recorded in [research.md](./research.md). Key decisions:

| Decision | Choice |
|----------|--------|
| Storage | Two columns on `items` table (Migration 012) |
| Notifications | New `WarrantyReminderService` wrapping `expo-notifications` |
| Lead times | 30 days before + expiry day at 09:00 |
| Notification body | Item name + location name |
| UX entry | Dedicated warranty card on item detail screen |
| Dashboard route | `/tools/warranty-tracker` |
| Deep-link | `Notifications.addNotificationResponseReceivedListener` in root layout |
| Warranty color | `#e09b3a` (amber — distinct from `PRIMARY` blue-gray and `LENDING` purple) |
