# Data Model: Lending Tracker

**Feature**: Lending  
**Created**: May 6, 2026  
**Scope**: MVP - local SQLite, offline-first

---

## Entity: Lending

Represents a single item lent to someone. Tracks the lending lifecycle from creation (ACTIVE) to completion (RETURNED).

### Attributes

| Attribute | Type | Required | Notes |
|-----------|------|----------|-------|
| `id` | UUID (TEXT) | Yes | Primary key, auto-generated |
| `item_id` | UUID (TEXT) | Yes | Foreign key to items.id |
| `borrower_name` | String | Yes | Name of person borrowing the item |
| `note` | String | No | Optional context about lending |
| `lent_at` | Date (ISO 8601) | Yes | Timestamp when lending was created |
| `returned_at` | Date (ISO 8601) | No | Timestamp when item was marked returned (null if ACTIVE) |
| `status` | Enum | Yes | 'ACTIVE' or 'RETURNED' |
| `created_at` | Date (ISO 8601) | Yes | Record creation timestamp |
| `updated_at` | Date (ISO 8601) | Yes | Record last modification timestamp |

### Constraints

- **Primary Key**: `id` (unique, not null)
- **Foreign Key**: `item_id` references `items.id` (no cascade on delete to preserve history)
- **Not Null**: `item_id`, `borrower_name`, `lent_at`, `status`, `created_at`, `updated_at`
- **Unique Constraint**: `(item_id, status)` where status='ACTIVE' (ensures one active lending per item)
- **Check**: `status IN ('ACTIVE', 'RETURNED')`
- **Check**: `returned_at IS NULL OR returned_at >= lent_at` (return date can't be before lend date)

### Indexes

| Index | Columns | Purpose |
|-------|---------|---------|
| PRIMARY | id | Primary key lookup |
| UNIQUE | (item_id, status='ACTIVE') | Enforce one active per item |
| QUERY_STATUS | (status, lent_at DESC) | List by status, sorted by date |
| QUERY_ITEM | (item_id) | Find lendings for deleted items |

## Relationships

### Lending ↔ Item

**Relationship Type**: Many-to-One (Multiple lendings can reference same item, but each lending references one item)

**Cardinality**:
- One Item → Many Lendings (one item can be lent multiple times, sequentially)
- One Lending → One Item (each lending is for exactly one item)

**Rules**:
- A lending is only valid if its referenced item exists (foreign key constraint)
- If item is deleted:
  - **Option A (Chosen for MVP)**: Lending record remains orphaned with item_id pointing to non-existent item
  - **Implication**: UI must handle case where item lookup returns null
  - **Fallback**: Display lending details without live item data (show borrower, date, status)

**Query Patterns**:
- "Get all lendings for item X": `SELECT * FROM lendings WHERE item_id = ?`
- "Check if item X has active lending": `SELECT * FROM lendings WHERE item_id = ? AND status = 'ACTIVE'`
- "Get all lendings with item details": JOIN lendings with items table (left join to handle orphans)

## State Machine: Lending Lifecycle

```
CREATE LENDING (initial state)
    ↓
ACTIVE (status = 'ACTIVE', returned_at = NULL)
    ├→ User receives item back
    ↓
RETURNED (status = 'RETURNED', returned_at = timestamp)
    ↓
END (record archived in history)
```

### Status Transitions

| From | To | Action | Conditions |
|------|----|---------|----|
| — | ACTIVE | createLending() | Item exists, no ACTIVE lending for item |
| ACTIVE | RETURNED | markAsReturned() | Lending.status == 'ACTIVE' |
| RETURNED | — | (no transition) | Records are immutable after return |

### State Queries

- **ACTIVE Lendings**: `SELECT * FROM lendings WHERE status = 'ACTIVE'`
- **RETURNED Lendings**: `SELECT * FROM lendings WHERE status = 'RETURNED'`
- **All Lendings**: `SELECT * FROM lendings` (no filter)
- **Sorted by Date**: `ORDER BY lent_at DESC`

## Data Validation Rules

### At Creation

| Field | Rule | Error Message |
|-------|------|------|
| borrower_name | Not empty, not whitespace | "Borrower name required" |
| item_id | Must exist in items table | "Item not found" |
| item_id | Must not have ACTIVE lending | "Item already lent out" |

### At Return

| Field | Rule | Error Message |
|-------|------|------|
| lending_id | Must exist | "Lending not found" |
| status | Must be ACTIVE | "Item already returned" |

### At Query

| Query | Rule |
|-------|------|
| Get ACTIVE | Return sorted by lent_at DESC (most recent first) |
| Get All | Return sorted by lent_at DESC (most recent first) |
| Get by Item | Handle orphaned items (item may not exist) |

## Timestamp Management

**All timestamps are ISO 8601 strings in SQLite**

### Generation

| Timestamp | Generated By | When |
|-----------|------|------|
| `lent_at` | LendingRepository.create() | On lending creation |
| `created_at` | LendingRepository.create() | On lending creation |
| `updated_at` | LendingRepository.create() | On lending creation (= created_at initially) |
| `returned_at` | LendingRepository.markAsReturned() | When marked as returned |
| `updated_at` | LendingRepository.markAsReturned() | When marked as returned |

**Rule**: Never create timestamps in Service or UI. Repository is the single source of truth.

## Query Operations

### Read Operations

```sql
-- Get all ACTIVE lendings
SELECT * FROM lendings WHERE status = 'ACTIVE' ORDER BY lent_at DESC

-- Get all RETURNED lendings
SELECT * FROM lendings WHERE status = 'RETURNED' ORDER BY lent_at DESC

-- Get all lendings (history)
SELECT * FROM lendings ORDER BY lent_at DESC

-- Get single lending
SELECT * FROM lendings WHERE id = ?

-- Check if item has active lending
SELECT * FROM lendings WHERE item_id = ? AND status = 'ACTIVE'

-- Get all lendings for a deleted item (orphaned records)
SELECT * FROM lendings WHERE item_id = ?
```

### Write Operations

```sql
-- Create lending
INSERT INTO lendings 
  (id, item_id, borrower_name, note, lent_at, returned_at, status, created_at, updated_at)
VALUES 
  (?, ?, ?, ?, ?, NULL, 'ACTIVE', ?, ?)

-- Mark as returned
UPDATE lendings 
SET status = 'RETURNED', returned_at = ?, updated_at = ?
WHERE id = ?
```

## Relationships with Existing Entities

### Lending → Item

- **Many-to-One**: Multiple lendings can reference the same item
- **Foreign Key**: `item_id` → `items.id`
- **On Delete Item**: Lending orphaned (no cascade delete)
- **UI Implication**: When displaying lending, lookup item by item_id; if null, show "[Item deleted]"

### Lending → User

- **Note**: No explicit User entity in MVP (single-user app)
- **Implicit**: All lendings belong to logged-in user (or device user)
- **Future**: If multi-user added, add user_id foreign key

### Lending ← Item doesn't have state

- **Important**: Item entity doesn't have "lent" state
- **Reasoning**: Item still belongs to original space/container even when lent
- **Query Pattern**: To find all lent items: `SELECT DISTINCT item_id FROM lendings WHERE status = 'ACTIVE'`

## Data Integrity Safeguards

| Risk | Safeguard |
|------|-----------|
| Duplicate ACTIVE per item | Unique index + service check |
| Invalid status | Check constraint + TypeScript enum |
| Missing borrower name | NOT NULL constraint + service validation |
| Returned before lent | Check constraint + logic prevents reversal |
| Orphaned item_id | FK without cascade + UI handles null |
| Lost history | No automatic cleanup, records immutable |

## Example: User Journey with Data

### Scenario: Lend Item to Bob

1. User taps "Lend Item"
2. Selects "Bicycle" (item_id = abc123) from Items list
3. Enters borrower name: "Bob"
4. Enters note: "Mountain bike - bring back by weekend"
5. Taps "Lend"

**Database Result**:
```sql
INSERT INTO lendings VALUES (
  id = 'lending-001',
  item_id = 'abc123',
  borrower_name = 'Bob',
  note = 'Mountain bike - bring back by weekend',
  lent_at = '2026-05-06T14:30:00Z',
  returned_at = NULL,
  status = 'ACTIVE',
  created_at = '2026-05-06T14:30:00Z',
  updated_at = '2026-05-06T14:30:00Z'
)
```

### Later: Mark as Returned

1. User navigates to Lending tab
2. Sees "Bicycle - Bob - May 6"
3. Taps item → Detail screen
4. Taps "Mark as Returned"

**Database Result**:
```sql
UPDATE lendings 
SET status = 'RETURNED', 
    returned_at = '2026-05-07T10:00:00Z',
    updated_at = '2026-05-07T10:00:00Z'
WHERE id = 'lending-001'
```

**Now in History**:
- Bicycle appears in "All Lendings" view
- Shows: "Bicycle - Bob - Lent: May 6, Returned: May 7"
- Status badge shows "RETURNED"

## Performance Considerations

| Operation | Index Used | Complexity | Notes |
|-----------|-----------|-----------|-------|
| List active lendings | (status, lent_at DESC) | O(n) | Fast with index |
| Check one active per item | (item_id, status) | O(1) | Unique constraint |
| Get lending detail | id (primary key) | O(1) | Direct lookup |
| List all lendings | (lent_at DESC) or table scan | O(n) | Fine for MVP |

**Scaling Considerations**:
- At 100s of lendings: Current indexes sufficient
- At 1000s of lendings: Consider pagination in UI
- At 10000s: Add date-range filtering
